SecretSifter — Privacy Policy

Effective date: March 11, 2026

Summary: SecretSifter does not collect, transmit, or share any data. All findings are stored exclusively in your local browser and never leave your device.

1. What SecretSifter Does

SecretSifter is a browser extension designed for security professionals and penetration testers. It intercepts network responses (JavaScript files, JSON APIs, XML responses, HTML pages, and WebSocket frames) in the active browser tab and scans them for exposed secrets such as API keys, tokens, passwords, and credentials using pattern-matching rules.

2. Data Collection

SecretSifter does not collect any personal data. Specifically:

  • No data is transmitted to any external server, including the developer's servers.
  • No analytics, telemetry, or usage tracking of any kind is performed.
  • No user accounts are created or required.
  • No browsing history is recorded beyond what is temporarily held in local browser storage for the current session.

3. Local Storage

SecretSifter stores the following data locally in your browser only (using chrome.storage.local / browser.storage.local):

  • Scan findings (secret matches detected during your browsing session)
  • User settings (CDN blocklist customizations, noise key list)
  • The list of hostnames you have enabled scanning on

This data never leaves your device. You can clear all stored findings at any time using the "Clear" button in the extension popup or DevTools panel.

4. Permissions Used

  • debugger — Used to attach to the Chrome DevTools Protocol (CDP) to intercept network responses in the active tab. This is required for full network coverage and is only activated when you explicitly enable scanning on a tab.
  • tabs — Used to associate scan findings with the correct browser tab.
  • storage — Used to persist findings and settings locally in the browser.
  • activeTab / scripting — Used to inject the network interceptor into the current tab's page context when scanning is enabled.
  • <all_urls> — Required so the extension can operate on any website the user chooses to scan. Scanning is always opt-in per tab.

5. Third-Party Services

SecretSifter does not use any third-party SDKs, analytics services, advertising networks, or external APIs. The extension operates entirely offline.

6. Children's Privacy

This extension is intended for security professionals and is not directed at children under the age of 13. We do not knowingly collect any information from children.

7. Changes to This Policy

If this privacy policy is updated, the effective date above will be revised. Significant changes will be noted in the extension's version release notes.

8. Contact

For questions about this privacy policy, please open an issue on the GitHub repository or contact the developer directly via GitHub.
